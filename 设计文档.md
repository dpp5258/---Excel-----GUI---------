# 日常提醒任务管理系统设计文档

## 文档信息
1. 项目名称：基于Excel存储的GUI日常任务提醒系统
2. 开发语言：Python
3. 存储载体：单文件 `任务清单.xlsx`（三Sheet：待办任务、已完成任务、归档任务）
4. 依赖库：PyQt6、openpyxl
5. 设计目标：支持GUI可视化管理任务，支持手动Excel编辑，双向数据同步，任务可标记完成/撤回、增删改查

---

# 一、需求分析

## 1.1 功能需求

### 基础存储需求
1. 单一Excel文件承载全部数据，分三张工作表：待办任务、已完成任务、归档任务
2. 三表字段完全统一，方便任务跨表迁移
3. 支持用户脱离程序，直接打开Excel手动修改所有任务数据
4. 程序可一键刷新，同步Excel内手动修改的数据
5. 每次程序写入前自动备份Excel到 `backup/` 文件夹，保留最近10份

### GUI操作需求
1. **新增任务**：自动生成全局唯一ID、自动填充创建时间，填写标题、内容、截止时间、优先级、备注
2. **修改任务**：双击任务行或点击编辑按钮，弹窗回填全部字段供编辑
3. **删除任务**：弹出确认框，确认后永久删除
4. **状态流转**：
   - 待办任务标记完成 → 弹窗填写完成情况 → 移动至【已完成任务】Sheet
   - 已完成任务撤回 → 移回【待办任务】Sheet，状态统一重置为「未开始」
5. **清空已完成**：将已完成任务全部迁移至【归档任务】Sheet，不丢失数据，需二次确认
6. **任务展示**：主排序按优先级（0-100）降序；同优先级按截止时间升序（越早越靠前）；截止时间相同按ID升序
7. **刷新功能**：重新读取Excel，以Excel数据覆盖内存，加载最新数据
8. **多级过期提醒**：根据截止时间与当前时间对比，行背景色区分：
   - 已过期（超过截止时间）：**深红色** `#8B0000`
   - 1天（24小时）内到期：**红色** `#DC143C`
   - 3天（72小时）内到期：**橙色** `#FF8C00`
   - 3天以上 / 无截止时间：**绿色** `#228B22`
9. **关键字搜索**：顶部搜索框，按任务标题实时过滤（不区分大小写）

### 数据约束需求
1. ID：全局自增数字，三表不重复
2. 优先级：数字区间 0~100，数值越大优先级越高
3. 完成状态枚举：`未开始`、`进行中`、`已完成`
4. 待办Sheet仅存放「未开始/进行中」任务；已完成Sheet仅存放「已完成」任务；归档Sheet仅存放「已完成」任务
5. 时间格式：严格 yyyy-MM-dd HH:mm，空字符串表示未设置，非法格式拒绝保存并提示修正

## 1.2 非功能需求
1. 轻量化：无需数据库，仅本地Excel文件存储，免安装服务
2. 双向同步：GUI操作实时写入Excel；Excel修改后程序可重载（刷新 = Excel覆盖内存，刷新前未保存的GUI操作将丢失）
3. 易用性：界面简洁，新手可快速上手；Excel表格自带筛选、排序功能
4. 兼容性：仅使用xlsx格式，兼容Windows/macOS自带Office/WPS
5. 容错性：启动自动校验Excel文件，文件缺失/表头损坏自动重建
6. 可追溯性：操作日志写入 `control.log`，记录增删改、刷新、异常等关键事件
7. 数据安全：每次写入前自动备份到 `backup/任务清单_yyyyMMdd_HHmmss.xlsx`

# 二、数据存储设计

## 2.1 文件结构
文件名：`任务清单.xlsx`

| 序号 | Sheet名称 | 用途 | 允许的任务状态 |
|------|----------|------|--------------|
| 1 | 待办任务 | 当前需要处理的任务 | 未开始、进行中 |
| 2 | 已完成任务 | 已完成但尚未归档的任务 | 已完成 |
| 3 | 归档任务 | 从已完成表归档的历史任务 | 已完成 |

## 2.2 统一数据表结构（三张表表头完全一致）

| 字段序号 | 字段名称 | 数据类型 | 约束规则 |
|--------|--------|--------|--------|
| 1 | ID | 整型数字 | 全局唯一自增，三表共用一套ID，不可重复 |
| 2 | 任务标题 | 字符串 | **非空**，任务简短名称 |
| 3 | 任务内容 | 字符串 | 可空，任务详细描述 |
| 4 | 创建时间 | 时间字符串 | 格式 yyyy-MM-dd HH:mm，新增任务程序自动生成 |
| 5 | 截止时间 | 时间字符串 | 格式 yyyy-MM-dd HH:mm，允许为空 |
| 6 | 优先级 | 整型数字 | 取值 0~100，数值越大优先级越高 |
| 7 | 备注 | 字符串 | 可空，附加补充信息 |
| 8 | 完成状态 | 枚举字符串 | 可选值：未开始 / 进行中 / 已完成 |
| 9 | 完成情况 | 字符串 | 待办任务可空；已完成/归档任务建议填写总结 |

## 2.3 数据表业务规则
1. **待办任务Sheet**：仅允许「未开始、进行中」两种状态，若读取到已完成状态自动迁移至完成表
2. **已完成任务Sheet**：所有行完成状态强制为「已完成」
3. **归档任务Sheet**：所有行完成状态强制为「已完成」，由清空已完成操作迁移写入
4. **任务迁移逻辑**：
   - 标记完成：从待办表删除该行 → 插入已完成表（状态改为已完成，补填完成情况）
   - 撤回任务：从已完成表删除该行 → 插入待办表（状态统一重置为「未开始」）
   - 归档任务：从已完成表删除该行 → 插入归档表（保留原数据和完成情况）
5. **ID生成规则**：读取三张表全部ID，取最大值+1作为新任务ID；无数据时ID=1
6. **ID去重策略**：加载时若发现ID冲突，保留ID较小的记录，冲突行重新分配新ID

# 三、系统分层架构设计

四层低耦合架构，界面、业务、数据完全解耦。

```
┌──────────────────────────────────┐
│  GUI层 (gui.py)                  │  PyQt6 界面，仅展示与交互
│  MainWindow + 3个Dialog         │
├──────────────────────────────────┤
│  控制层 (controller.py)          │  业务逻辑调度，连接GUI与存储层
│  TaskController                 │
│  todo_tasks: dict[int, Task]    │
│  done_tasks: dict[int, Task]    │
│  archive_tasks: dict[int, Task] │
├──────────────────────────────────┤
│  模型层 (model.py)               │  任务实体(dataclass) + 日志工具
│  Task / DeadlineStatus / Logger │
├──────────────────────────────────┤
│  存储层 (storage.py)             │  Excel读写、备份、校验
│  ExcelStorage                   │
└──────────────────────────────────┘
```

## 3.1 存储层（Excel操作模块）— storage.py
负责所有文件读写、文件初始化、表头校验、数据加载、数据写入、自动备份。

核心功能：
1. **文件初始化**：文件不存在 / 表头损坏时，自动重建Excel与三Sheet表头
2. **读取全部数据**：分别读取待办、已完成、归档三张表，转为内存任务列表
3. **写入数据**：新增、更新、删除、迁移任务后覆盖写入Excel
4. **自动备份**：每次写入前复制当前 `任务清单.xlsx` 到 `backup/任务清单_yyyyMMdd_HHmmss.xlsx`，保留最近10份
5. **文件占用检测**：写入失败（PermissionError）时抛出明确异常，由上层处理
6. **工具方法**：获取全局最大ID、校验表头完整性

## 3.2 模型层（模型实体与工具）— model.py
- **Task类**（使用 `@dataclass`）：
  - 属性：id、title、content、create_time、deadline、priority、note、status、finish_desc
  - 静态方法：`row_to_task(row)`、`task_to_row(task)`
  - 实例方法：`deadline_status()` → 返回 `DeadlineStatus` 枚举（expired / urgent / warning / safe）

- **DeadlineStatus枚举**：
  ```python
  class DeadlineStatus(Enum):
      EXPIRED = "expired"   # 已过期 → 深红色 #8B0000
      URGENT = "urgent"     # 1天内  → 红色   #DC143C
      WARNING = "warning"   # 3天内  → 橙色   #FF8C00
      SAFE = "safe"         # 安全   → 绿色   #228B22
  ```

- **Logger工具**：封装 Python logging，输出到 `control.log`，格式 `[yyyy-MM-dd HH:mm:ss] LEVEL message`

## 3.3 业务控制层（调度核心）— controller.py
连接GUI与存储层，封装所有业务逻辑，GUI不直接操作Excel。内存数据结构使用 `dict[int, Task]`，key为任务ID，便于O(1)查找。

核心接口：

| 接口 | 说明 |
|------|------|
| `load_all_tasks()` | 加载待办、已完成、归档全部任务到内存dict |
| `create_task(title, content, deadline, priority, note)` | 新增待办任务，返回Task |
| `update_task(task_id, **fields)` | 修改已有任务全部字段 |
| `delete_task(task_id)` | 删除指定ID任务（调用前GUI层先弹确认框） |
| `finish_task(task_id, finish_desc)` | 待办→已完成，状态改为已完成 |
| `revert_task(task_id)` | 已完成→待办，状态重置为「未开始」，完成情况清空 |
| `archive_all_done()` | 已完成全部迁移至归档表，返回归档数量 |
| `refresh()` | 重新读取Excel覆盖内存（刷新前未保存的操作将丢失） |
| `search(keyword)` | 按标题关键字搜索，返回匹配的待办和已完成任务列表 |

## 3.4 GUI界面层（可视化交互）— gui.py
基于PyQt6开发，仅负责展示与接收用户操作，所有业务请求交给控制层。

界面布局（单窗口）：
```
┌──────────────────────────────────────────────┐
│ [新增任务] [刷新] [清空已完成] [退出]           │  ← 工具栏
│ 🔍 搜索: [________________]                   │  ← 搜索框
├──────────────────────┬───────────────────────┤
│ ◉ 待办任务            │ ○ 已完成任务           │  ← QTabWidget 两选项卡
│                      │                       │
│ [深红] 过期任务A      │ ✓ 任务D  [撤回]       │
│ [红色] 临期任务B      │ ✓ 任务E  [撤回]       │
│ [橙色] 即将任务C      │ ✓ 任务F  [撤回]       │
│ [绿色] 安全任务...     │                       │
│                      │                       │
│     [编辑] [删除] [✓完成] │  [编辑] [删除]       │  ← 选中任务后可用
└──────────────────────┴───────────────────────┘
```

核心组件：
| 组件 | 说明 |
|------|------|
| `MainWindow` | 主窗口：工具栏 + 搜索框 + 选项卡 + 状态栏 |
| `TaskDialog(QDialog)` | 新增/编辑弹窗：标题(必填)、内容、截止时间、优先级(0-100)、备注 |
| `FinishDialog(QDialog)` | 完成确认弹窗：任务概要 + 完成情况输入框 |
| `QMessageBox` | 删除确认、归档确认、错误提示 |

界面联动规则：
1. 程序启动 → 控制层加载Excel数据 → 渲染两选项卡列表 → 计算截止时间状态着色
2. 用户点击操作 → 调用控制层对应方法 → 控制层调用存储层写入Excel → 刷新界面
3. 用户手动修改Excel后点击「刷新」→ 重载文件，Excel数据覆盖内存 → 更新界面
4. 搜索框输入文字 → 实时过滤两列表显示
5. 文件写入失败（PermissionError）→ 弹窗提示「请关闭Excel/WPS后重试」

# 四、核心业务流程设计

## 4.1 程序启动流程
1. 创建 `backup/` 目录（如不存在）
2. 初始化存储模块，检查 `任务清单.xlsx`
3. 文件不存在/表头错误 → 自动重建Excel三Sheet模板
4. 控制层读取三张表所有数据 → 转为 `dict[int, Task]` 存入内存
5. GUI加载内存数据，渲染待办、已完成双列表
6. 计算每个任务的 `deadline_status()`，应用对应行颜色

## 4.2 新增任务流程
1. 用户点击「新增任务」→ 弹出 TaskDialog
2. 输入标题（必填）、内容、截止时间（可选）、优先级（0-100，默认0）、备注（可选）
3. 点击确认 → **校验**：
   - 标题非空 ✓
   - 优先级为0-100整数 ✓
   - 截止时间为空或符合 yyyy-MM-dd HH:mm ✓
   - 任一失败 → 弹窗提示具体错误，留在弹窗
4. 校验通过 → 控制层获取全局最大ID+1 → 自动生成创建时间 → 状态默认「未开始」
5. 存储层写入「待办任务」Sheet（写入前自动备份）
6. 内存更新，界面刷新

## 4.3 标记任务完成流程
1. 用户选中待办任务 → 点击「完成」→ 弹出 FinishDialog
2. 弹窗显示任务标题、创建时间等概要 → 用户填写完成情况
3. 确认 → 控制层将该任务从待办 dict 移除 → 状态改为「已完成」→ 写入完成情况
4. 存储层：待办表删除对应行 → 已完成表新增该行（写入前自动备份）
5. 界面：左侧待办移除 → 右侧已完成新增该行

## 4.4 撤回已完成任务流程
1. 用户选中已完成任务 → 点击「撤回」
2. 控制层将该任务从已完成 dict 移除 → 状态改为「未开始」→ 完成情况清空
3. 存储层：已完成表删除对应行 → 待办表新增该行（写入前自动备份）
4. 界面：右侧已完成移除 → 左侧待办新增该行

## 4.5 归档已完成任务流程
1. 用户点击「清空已完成」→ 弹出确认框「将 N 条已完成任务归档到归档表？可手动从Excel找回。」
2. 确认 → 控制层将所有已完成任务迁移至归档 dict
3. 存储层：已完成表所有行删除 → 归档表新增这些行（写入前自动备份）
4. 界面：右侧已完成列表清空

## 4.6 手动修改Excel同步流程（刷新）
1. 用户在程序运行期间或关闭程序后，手动编辑了Excel
2. 点击「刷新」按钮
3. 存储层重新读取三张Sheet全部数据
4. 控制层以Excel数据**覆盖**内存三组 dict（刷新前未保存的GUI操作将丢失）
5. GUI清空原有列表，重新渲染最新数据，重新计算过期颜色

## 4.7 编辑任务流程
1. 选中列表任务 → 点击「编辑」或双击行 → 弹出 TaskDialog 回填全部字段
2. 修改信息 → 确认 → 同新增流程的校验规则
3. 校验通过 → 控制层更新内存对应的 Task 对象
4. 存储层定位对应ID行，覆盖Excel单元格数据（写入前自动备份）
5. 界面刷新展示修改后内容

## 4.8 删除任务流程
1. 选中任务 → 点击「删除」→ 弹出确认框「确定删除任务"[标题]"？此操作不可恢复。」
2. 确认 → 控制层从对应 dict 移除该任务
3. 存储层在对应Sheet中删除该行（写入前自动备份）
4. 界面移除该行

## 4.9 搜索流程
1. 用户在搜索框输入关键字
2. 实时过滤：标题包含关键字（不区分大小写）的任务显示，其余隐藏
3. 清空搜索框 → 恢复全部显示

关键实现要点：
- `write_all()` 调用前先执行 `backup()`

- `get_max_id()` 遍历三张Sheet取最大ID

# 六、异常与容错设计

| # | 异常场景 | 处理策略 |
|---|---------|---------|
| 1 | Excel文件被占用 | PermissionError捕获 → 弹窗「请关闭Excel/WPS后重试」 |
| 2 | 表头缺失/字段篡改 | 启动时 `validate_headers()` 检测 → 自动重建标准表头，保留原有有效数据 |
| 3 | 优先级输入非数字 | 弹窗拦截「优先级请输入0-100的整数」 |
| 4 | 优先级超出范围 | 弹窗拦截「优先级范围为0-100」 |
| 5 | 完成状态非法值 | 读取时自动修正：异常状态任务移入待办表，状态改为「未开始」 |
| 6 | ID重复 | 加载时自动去重，保留ID较小的记录，冲突行重新分配新ID |
| 7 | 时间格式错误 | 弹窗提示「时间格式应为 yyyy-MM-dd HH:mm」，拒绝保存 |
| 8 | 标题为空 | GUI层拦截，不允许提交 |
| 9 | backup/ 目录不存在 | 启动时自动创建 |
| 10 | 备份写入失败 | 不阻断主流程，日志记录警告 |

# 七、截止时间色彩方案

| 状态 | 判断条件 | 颜色名 | 色值 |
|------|---------|--------|------|
| 已过期 EXPIRED | `deadline < now` 且 deadline 非空 | 深红 DarkRed | `#8B0000` |
| 紧急 URGENT | `now < deadline <= now + 1天` | 红色 Crimson | `#DC143C` |
| 预警 WARNING | `now + 1天 < deadline <= now + 3天` | 橙色 DarkOrange | `#FF8C00` |
| 安全 SAFE | `deadline > now + 3天` 或 deadline 为空 | 绿色 ForestGreen | `#228B22` |

应用方式：QTableWidget 行背景色，由 `Task.deadline_status()` 驱动，`MainWindow._apply_row_color()` 执行着色。

# 九、部署与使用说明
1. 环境依赖安装：`pip install pyqt6 openpyxl`
2. 运行入口：`python gui.py`，自动生成 `任务清单.xlsx` 和 `backup/` 目录
3. 手动编辑规范：
   - 不可删除、修改第一行表头文字
   - 优先级仅填写0-100数字
   - 时间统一 yyyy-MM-dd HH:mm 格式
   - 待办Sheet不要手动写入「已完成」状态
4. 注意事项：GUI读写Excel前请关闭WPS/Office，避免文件锁定报错
